# LB3_DS_Blockchain — Децентралізована соціальна мережа на блокчейні

## Огляд та призначення системи

**LB3_DS_Blockchain** — це реалізація децентралізованої соціальної мережи на основі технології блокчейну на C# (.NET 8). Система дозволяє користувачам безпечно ділитися постами, коментарями, лайками та підписуватися один на одного з гарантованою цілісністю та автентичністю всіх операцій.

### Основна мета:
- **Безпечна комунікація** — всі операції криптографічно підписані
- **Незмінна історія** — усі пости та коментарі записуються в незмінний реєстр
- **Децентралізована** — немає центрального сервера для контролю
- **Прозорість** — вся історія публічна та перевіряєма

## Функції продукту (короткий опис)

### 1. Управління користувачами

**Основні операції:**
- `GenAccount()` — створення нового облікового запису з RSA-2048 ключовою парою
- `RegisterUser()` — реєстрація користувача в мережі з імям та біографією
- `AddKeyPairToWallet()` — додавання нової ключової пари до гаманця користувача
- `UpdateBalance()` — оновлення балансу користувача
- `SignData()` — підпис повідомлення приватним ключем
- `GetBalance()` — отримання поточного балансу

**Дані користувача:**
- Унікальний ID (SHA256 хеш публічного ключа)
- Гаманець з ключовими парами (мінімум 1)
- Баланс (кількість цифрових активів)
- Список гаманців для підписання операцій

---

### 2. Управління профілем

**Основні операції:**
- `SetUsername()` — встановлення або змінення імені користувача
- `SetBio()` — редагування біографії профілю
- `Follow()` / `Unfollow()` — управління підписками
- `AddPost()` — добавлення ID поста до списку користувача
- `AddReputation()` — накопичення репутаційних балів

**Дані профілю:**
- Ім'я користувача (унікальне, не порожне)
- Біографія (довільний текст)
- Список підписників
- Список того, на кого підписаний
- Список власних постів
- Дата реєстрації
- Репутація (бали за активність)

---

### 3. Управління контентом (пости та коментарі)

**Пости:**
- `CreatePost()` — створення поста з контентом та автором
- `AddLike()` / `RemoveLike()` — управління лайками
- `GetContent()` — отримання контенту поста
- Автоматична генерація SHA256 хеша контенту

**Коментарі:**
- `CreateComment()` — додавання коментара до конкретного поста
- `AddLike()` / `RemoveLike()` — управління лайками на коментарях
- Посилання на батьківський пост
- Автор та часова мітка

**Дані контенту:**
- Унікальний ID (SHA256 хеш)
- Автор (Account)
- Контент (довільний текст)
- Часова мітка створення (UTC)
- Лічильник лайків
- Хеш контенту для перевірки цілісності

---

### 4. Криптографічні операції

**Управління ключами:**
- `GenKeyPair()` — генерація RSA-2048 ключової пари
- `GetPrivateKey()` — отримання приватного ключа (захищено)
- `PublicKey` — публічний ключ (доступний)

**Хешування:**
- `ToSHA256()` — криптографічне хешування SHA-256 (256-біт)
- `ToSHA1()` — хешування SHA-1 (160-біт, застаріла)

**Цифрові підписи:**
- `SignData()` — підпис повідомлення приватним ключем (RSA-2048)
- `VerifySignature()` — верифікація підпису за публічним ключем
- Алгоритм хешування: SHA1 (для сумісності з RSA)

**Безпека операцій:**
- Кожна операція содержит підпис автора
- Верифікація підпису перед додаванням в блокчейн
- Неможливо підробити операцію без приватного ключа

---

### 5. Управління блокчейном

**Операції:**
- `CreateOperation()` — створення операції (Post, Comment, Like, Follow, Share)
- `VerifyOperation()` — перевірка підпису та баланса операції
- `CreateTransaction()` — групування операцій у транзакцію
- `CreateBlock()` — формування блока з транзакцій
- `ValidateBlock()` — комплексна валідація блока
- `GetBlockHistory()` — отримання всієї історії блокчейну

**Типи операцій:**
- `CreatePost` — створення нового поста
- `CreateComment` — додавання коментара
- `Like` — лайк до контенту
- `Follow` — підписка на користувача
- `Share` — поділитися постом

**Процес валідації блока:**
1. Перевірка посилання на попередній блок (prevHash)
2. Перевірка унікальності кожної транзакції
3. Перевірка цифрового підпису кожної операції
4. Контроль баланса (якщо застосовується)
5. Контроль конфліктів між операціями в транзакції
